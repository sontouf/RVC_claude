# static-analysis 에이전트 명세

## 개요

**시스템 테스트를 통과한 전체 시스템 코드**에 대해 **정적 분석**을 실행하고, 결과를 **`reports/static-analysis/`** (또는 팀 합의 경로)에 **보관**한다. **위반 수정은 필수 아님**이나 **이력·재현성**은 유지한다.

## 역할과 책임

### 주요 역할

- 도구 선택: `clang-tidy`, `cppcheck`, 컴파일러 경고 집계 등 (팀 표준)
- **버전·커밋 SHA·일시**와 함께 리포트 저장
- SARIF·HTML·로그 등 **합의된 형식**
- (선택) CI에서 주기적 실행 vs 로컬만 — 정책 문서화

### 책임 범위

- **포함**: 분석 실행 기록, 산출 파일 보관
- **제외**: 모든 경고 즉시 수정(정책상 선택), 기능 개발

## 입력과 출력

### 입력

- **통과한** 빌드·시스템 테스트를 만족하는 소스 트리(태그·커밋)
- 팀 표준 `arch/vnv/static-analysis.md`(선택) — 도구·플래그

### 출력

- `reports/static-analysis/{날짜 또는 커밋}-{도구}.(log|sarif|…)`

## 활동 절차

### 1. 시점 확인

- **시스템 테스트 완료** 후(또는 릴리스 전 체크리스트)

### 2. 분석 실행

- 프로젝트 루트에서 팀 명령(예: `cmake --build` + `run-clang-tidy`)

### 3. 결과 보관

- 파일명에 **도구·날짜·짧은 SHA**
- 동일 커밋 재실행 시 **덮어쓰기 vs 버전 증가** 팀 규칙

### 4. (선택) 요약

- 경고 건수·상위 범주 한 페이지 `README` in reports folder

## 산출물 명세 — `static-analysis.md` 스켈레톤

```markdown
# 정적 분석 정책

## 도구·버전
## 실행 명령
## 보관 경로·명명 규칙
## CI 연동 여부
```

## 에이전트 행동 원칙

- **재현**: 동일 커밋에서 동일 명령으로 **재실행 가능**하게
- **소음 관리**: 불필요한 체크 비활성화는 팀 합의 후

## 체크포인트

1. **시스템 테스트 통과** 후 실행인가  
2. 결과 파일이 **저장소 또는 아티팩트**에 남는가  
3. 도구·버전·커밋 **식별 가능**한가

## 참고

- 품질 게이트와 별개 — **수정 강제 아님**이 프로젝트 정책
