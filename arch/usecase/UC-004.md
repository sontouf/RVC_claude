# UC-004: Escape a Triple-Blocked Enclosure

## Name

**UC-004 — Escape a Triple-Blocked Enclosure**

전·좌·우가 모두 막혔을 때 RVC는 후진하고, 후진 후 측면 중 한쪽이 열리면 그쪽으로 회전 후 전진 청소를 재개한다. 모든 시퀀스는 `CleaningCoordinator`가 단일 오케스트레이션으로 수행한다.

## Actor

| Role | Actor |
|------|-------|
| Primary | Sensor Subsystem |
| Secondary | Actuator Subsystem |

## Pre-Requisites

- UC-001을 통해 세션이 Running 상태다.
- `front=true && left=true && right=true`로 보고된다.

## Typical Courses of Events

1. 시스템이 한 tick을 시작하고 센서를 읽는다.
2. 삼면 막힘(`front && left && right`)을 인식한다.
3. 시스템은 액추에이터에 **stop cleaning motion**을 보낸 뒤 **drive backward**를 보낸다(이번 tick). 단 `maxBackoffTicks == 0`이면 backward는 발신하지 않는다.
4. 다음 tick에서 다시 센서를 읽는다.
5. **측면 중 한 쪽이 열렸음**을 확인한다(예: `left=false`). **전방만 열린 경우는 종료 조건이 아니며 후진을 계속한다** — 전방이 열렸다고 전진하면 같은 함정으로 다시 진입하기 때문(FR-004).
6. 측면이 열린 시점에 시스템은 그 방향으로 **turn(direction)**을 보낸다. 양쪽 모두 열렸으면 **left** 우선(FR-003 deterministic 정책 일관성). 이 tick에서는 **전진 명령을 발신하지 않는다**.
7. 다음 tick부터 시스템은 **drive forward**로 청소를 재개한다(UC-002).

## Alternative Courses of Events

- **A1.** 후진 후에도 양 측면이 모두 막혀 있으면(전방이 열렸든 닫혔든 무관) 시스템은 다시 후진을 시도한다(반복). `ControllerConfig::maxBackoffTicks`는 **안전 상한**으로 사용되며, 도달 시 정지 명령을 보내고 다음 tick에서 재시도한다.
- **A2.** `maxBackoffTicks == 0` 이면 후진 자체를 시도하지 않고 정지 상태를 유지한다(센서가 변할 때까지 대기).
- **A3.** 후진 도중 후방 안전 센서(향후 확장)가 추가되면 안전 거리를 두고 정지하도록 정책을 보강한다 — 현 버전에서는 후방 센서를 가정하지 않는다.

## Exceptional Courses of Events

- **E1.** 후진 명령이 거부되면 다음 tick에서 정지 명령을 보내고 재시도한다(NFR-SAFE-001).
- **E2.** 센서가 갑자기 정상으로 돌아오면 본 UC는 종료되고 UC-002로 복귀한다.

## 구현 노트 (정책 보조)

- `NavigationPolicy::plan_escape_enclosure`는 **단일 회전·전진의 보조 시퀀스 스텁**이며, 전체 후진→회전→전진 시퀀스를 대체하지 않는다(`reproducibility` RULE §3). 코디네이터는 본 UC의 단계 전이를 직접 소유한다.
