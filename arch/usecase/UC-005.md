# UC-005: Boost Cleaning Power on Dust

## Name

**UC-005 — Boost Cleaning Power on Dust**

먼지 센서가 true를 보고하면 청소 파워를 한 단계 상승시키고, `ControllerConfig::dustBoostTicks` 만료 시 원래 파워로 자동 복귀한다.

## Actor

| Role | Actor |
|------|-------|
| Primary | Sensor Subsystem |
| Secondary | Actuator Subsystem |

## Pre-Requisites

- UC-001을 통해 세션이 Running 상태다.
- `dust=true`가 한 tick 이상 보고된다.

## Typical Courses of Events

1. 시스템이 한 tick을 시작하고 센서를 읽는다.
2. `dust=true`임을 확인한다.
3. 시스템은 청소 파워를 nominal → boosted로 상승시키고 액추에이터에 **set power = boosted**를 보낸다.
4. 부스트 타이머를 `dustBoostTicks` 값으로 초기화한다.
5. 매 tick마다 타이머를 1 감소시키며, 이 동안 `dust=true`가 다시 들어오면 타이머를 **다시 `dustBoostTicks`로 재설정**(debounce/extend).
6. 타이머가 0에 도달하면 시스템은 **set power = nominal**을 보내고 부스트 상태를 종료한다.

## Alternative Courses of Events

- **A1.** 부스트 중에 회피(UC-003) 또는 탈출(UC-004)이 트리거되어도 청소 파워는 부스트 값을 유지한다(주행 명령만 영향).
- **A2.** 세션이 정지되면(UC-001 stop) 부스트 타이머는 리셋되고 파워는 0이 된다.

## Exceptional Courses of Events

- **E1.** `dustBoostTicks <= 0` 으로 잘못 설정된 경우 시스템은 부스트를 적용하지 않고 nominal 파워를 유지한다(가드).
- **E2.** 액추에이터가 파워 설정 명령을 거부하면 다음 tick에서 재시도한다.
