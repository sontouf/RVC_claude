# UC-001: Start and Stop a Cleaning Session

## Name

**UC-001 — Start and Stop a Cleaning Session**

세션을 시작하면 RVC는 자동 청소 루프를 활성화하고, 정지하면 모든 액추에이터 명령을 차단·정지한다.

## Actor

| Role | Actor |
|------|-------|
| Primary | Owner (User) |
| Secondary | Actuator Subsystem, Sensor Subsystem |

## Pre-Requisites

- 컨트롤러가 정상적으로 초기화되어 있다(`ControllerConfig` 로드 완료).
- 외부 센서·액추에이터 포트가 부착되어 있다.
- 이전 세션이 종료(혹은 미시작) 상태다.

## Typical Courses of Events

1. Owner가 시스템에 `startSession()` 입력을 보낸다.
2. 시스템은 내부 상태를 **Running**으로 전이하고 청소 파워를 기본값으로 설정한다.
3. 매 tick마다 시스템은 센서를 읽고 정책에 따라 액추에이터에 주행·청소 명령을 전송한다.
4. Owner가 `stopSession()` 입력을 보낸다.
5. 시스템은 청소 파워를 0으로 낮추고, 액추에이터에 정지 명령을 보낸 뒤 내부 상태를 **Stopped**로 전이한다.
6. 이후 tick은 no-op(액추에이터 호출 없음)이다.

## Alternative Courses of Events

- **A1.** 세션 Running 중 다시 `startSession()`을 받으면 시스템은 멱등(idempotent)으로 처리하고 상태를 변경하지 않는다.
- **A2.** Stopped 상태에서 `stopSession()`을 다시 받아도 멱등 처리한다.

## Exceptional Courses of Events

- **E1.** 센서 포트가 일시적으로 응답하지 않으면 시스템은 안전을 위해 **정지 명령**을 보내고 다음 tick에서 재시도한다(NFR-SAFE-001).
- **E2.** 액추에이터 포트가 명령을 거부하면 시스템은 해당 tick의 실패를 로그 카운터에 기록하고 다음 tick에서 정상 흐름을 시도한다.
